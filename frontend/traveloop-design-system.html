8Accessibility
Accessibility

A11y Standards

Traveloop targets WCAG 2.1 Level AA compliance. Color contrast, keyboard navigation, focus management, and screen reader support are non-negotiable.

Color Contrast
All text must meet 4.5:1 contrast ratio (AA). Large text (18px+) requires 3:1. Never convey info through color alone.
Keyboard Navigation
All interactive elements must be focusable. Tab order follows visual flow. Focus ring: 3px Flame-400 with 2px offset, no removal via CSS outline:none.
Touch Targets
Minimum 44×44px touch targets on mobile. Buttons: 44px height. Icon-only buttons must have aria-label. Spacing between targets: min 8px.
Screen Readers
Use semantic HTML. All images need alt text. Forms need associated labels. Modals trap focus and announce to screen readers via role="dialog".
Form Errors
Error messages appear inline below the field. Use role="alert" for dynamic errors. Never rely solely on red color — include an icon and text description.
Multi-language
Set lang attribute per page. Use logical properties (margin-inline) over left/right. Font DM Sans supports 200+ Latin-based languages including all LTR scripts.

✓ Do

Use Flame-500 (#F04E00) or darker for text on white backgrounds. Always pair error states with an error icon AND text, not just a red border. Use aria-label on all icon-only buttons like "Add Stop" or "Delete Trip".

✗ Don't

Never use Flame-400 (#FF6B2B) for body text on white — it fails AA. Don't remove focus rings with outline:none without providing a visible alternative. Never use placeholder text as a label substitute.
